# Правила разработки Siemensbot

## Версионирование промптов

- Промпты хранятся только как отдельные версионные файлы: `prompts/<тип>/<версия>.md`.
- Имя файла без `.md` должно совпадать с идентификатором версии и `run-id`, под которым эта версия тестируется.
- Нельзя изменять файл промпта, который уже был протестирован или указан в `prompts/active.json`. Для следующего изменения нужно скопировать последнюю версию в файл с новым идентификатором и менять только его.
- Candidate-файлы также создаются в `prompts/<тип>/`, а не во временной папке и не поверх активной версии.
- После принятия новой версии нужно переключить соответствующее значение в `prompts/active.json`. Код не должен содержать захардкоженное имя конкретного версионного файла.
- Старые версии не удаляются и не переименовываются: они нужны для обычного файлового diff и воспроизводимости eval-прогонов.
- Для sales eval следует явно передавать версионный файл через `--prompt`; хэш и путь сохраняются в `run.json`.
